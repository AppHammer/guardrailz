# Pre-Deployment Checklist

Use this checklist before each PyPI release.

## Code Quality
- [ ] All tests pass: `python test_guardrailz_testdata.py`
- [ ] No syntax errors or linting issues
- [ ] Examples work correctly
- [ ] README.md is accurate and up-to-date

## Version & Metadata
- [ ] Version number updated in `pyproject.toml`
- [ ] Version follows semantic versioning (MAJOR.MINOR.PATCH)
- [ ] CHANGELOG or release notes prepared (if applicable)
- [ ] GitHub repository URL is correct in `pyproject.toml`

## Security
- [ ] No API keys or secrets in code
- [ ] No sensitive data in test files
- [ ] `.gitignore` is properly configured
- [ ] Environment variables documented

## Documentation
- [ ] README.md installation instructions are correct
- [ ] API documentation is current
- [ ] CLI usage examples are accurate
- [ ] License file (LICENSE) is present

## Build & Package
- [ ] Clean build: `rm -rf dist/ build/ *.egg-info`
- [ ] Build succeeds: `python -m build`
- [ ] Package checks pass: `twine check dist/*`
- [ ] Package contents verified: `tar -tzf dist/guardrailz-*.tar.gz`

## Testing
- [ ] Install package locally: `pip install dist/*.whl`
- [ ] Import works: `python -c "import guardrailz"`
- [ ] CLI works: `guardrailz --help` (if applicable)
- [ ] Uninstall test version: `pip uninstall guardrailz -y`

## PyPI Accounts
- [ ] PyPI account created: https://pypi.org/
- [ ] TestPyPI account created: https://test.pypi.org/
- [ ] API tokens generated and saved securely

## First Deployment
- [ ] Test on TestPyPI first
- [ ] Verify TestPyPI installation works
- [ ] Upload to production PyPI
- [ ] Verify production installation: `pip install guardrailz`

## Post-Deployment
- [ ] Tag release in git: `git tag vX.Y.Z`
- [ ] Push tags: `git push origin vX.Y.Z`
- [ ] Create GitHub release
- [ ] Verify package on PyPI: https://pypi.org/project/guardrailz/
- [ ] Test installation from PyPI: `pip install guardrailz`

## GitHub Actions (Optional)
- [ ] `PYPI_API_TOKEN` added to GitHub Secrets
- [ ] Workflows tested and working
- [ ] Release automation configured

---

## Quick Command Reference

```bash
# Clean and build
rm -rf dist/ build/ *.egg-info
python -m build

# Check
twine check dist/*

# Test on TestPyPI
twine upload --repository testpypi dist/*

# Deploy to PyPI
twine upload dist/*

# Tag release
git tag v0.1.0
git push origin v0.1.0
```
